# Mission: TypeScript PR Review

## Why

Review AI-generated TypeScript pull requests with enough confidence to catch runtime bugs, unsafe type changes, and unclear contracts before they land.

## Success looks like

- Explain what changed in a small TypeScript diff without repeating the code line by line.
- Identify when a type annotation hides a runtime validation gap.
- Leave a concise review comment that asks for the smallest useful fix.

## Constraints

- Lessons should stay practical and code-first.
- Prefer small examples that fit in one screen.
- Assume the learner knows JavaScript basics but wants sharper TypeScript judgment.

## Out of scope

- Advanced type gymnastics.
- Library-specific framework patterns unless they affect review judgment.
